# DB 설명 
1. Users (사용자): 회원 정보

2. Themes (주제): 게임 카테고리 (예: 음식, 동물, 직업)

3. Keywords (제시어): 실제 게임 정답 단어 
    (예: 김치찌개, 사자, 개발자)

4. GameHistory (게임 기록): 한 판의 게임에 대한 요약 정보 
    (방 정보, 승리 팀, 플레이 날짜)

5. GameResult (상세 전적): 유저 개인별 승패 및 역할 기록 
    (User와 GameHistory의 연결 고리)

------------------------

1. Users (사용자 테이블)
    - 가장 기본이 되는 테이블입니다.
    - nickname은 게임 내에서 보여질 이름입니다.

2. Themes (주제 카테고리)
    - 단어들을 그룹화하는 상위 개념입니다.
    - 예: id=1(음식), id=2(동물)

3. Keywords (제시어)
    - theme_id를 외래키(FK)로 가집니다. (1:N 관계)
    - 라이어 게임의 핵심 콘텐츠인 단어들이 저장됩니다.

4. GameHistory (게임 한 판의 기록)
    - 게임이 언제 시작해서 언제 끝났는지, 어떤 주제로 했는지 저장합니다.

    - 이 테이블에는 "누가 참여했는지"는 넣지 않습니다. (그건 1:N 문제 때문)

5. GameResult (참여자별 상세 기록)
    - 가장 중요한 테이블입니다.
    - game_id와 user_id를 모두 FK로 가집니다.

    - 해당 유저가 그 판에서 
    '라이어'였는지(role), '이겼는지(is_win)'를 기록합니다.
